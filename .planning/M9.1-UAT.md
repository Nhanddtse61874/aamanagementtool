## The 3 click-throughs

### G3 — Task List bands adaptively (TL-12)
1. Open **Task List**. In the team filter, check **one** team → rows are banded by **Project** (headers = project names, ARCS/ARMS/…), as today.
2. Now check **two or more** teams → the band headers switch to **team names** (alphabetical); a backlog with no team falls under a **"—"** band at the bottom.
3. Back to one team → headers return to project bands.
**Expected:** >1 team = team-name headers; 1 team = project headers. (Matches the old WPF app.)

### G6 — Daily Report picker scoped to the active team (DR-11)
1. Make sure your **active team** is set (sidebar switcher).
2. Open **Daily Report → Input**, click the **backlog picker**.
**Expected:** the picker lists **only the active team's** backlogs — no other team's, and not the hidden `DEFAULT`. (Before M9.1 it showed backlogs from *all* your teams.)

### G10 — Default task deactivate → reactivate round-trip (SET-05)  *(admin)*
1. Open **Settings → default tasks** (Workflow section). Each row now has an **Activate/Deactivate** button (no more one-way "Remove").
2. **Deactivate** one → it stays visible, labelled **"(inactive)"**.
3. **Activate** it again → it returns to active.
4. *(optional)* Check **Log Work** reflects the reactivated task after the default-task sync.
**Expected:** deactivation is **reversible** — the task never disappears for good. *(Writes to the live DB, but the round-trip returns it to the original state.)*
