## Margin notes (not verdicts on this section's claims)

1. **A10 row 19 (`ShowTeamColumn`, PARTIAL) looks understated.** The auditor grepped for `showTeamColumn`
   in templates and found none — true, but the Task List re-derives the same rule locally:
   `task-list.component.ts:129 teamMode = (this.teamIds()?.length ?? 0) > 1`, fed into
   `:130 bands = groupRows(this.rows(), this.teamMode(), this.teamNames())`. So Task List *does* switch to
   team banding when ≥2 teams are checked. The remaining true gap is narrower than written (Daily Board
   badge; the shared `showTeamColumn` signal having no consumer). PARTIAL is the safe direction, so I am
   not moving it — only flagging that it should not be re-scoped as "no team indicator anywhere".
2. The web Task List's `mutate()` drops re-entrant writes (`task-list.component.ts:249`), but the picker's
   checkbox is `[disabled]="saving()"` (`tag-picker.component.html:33`), so a toggle cannot be silently
   lost during an in-flight save. Checked because a dropped toggle would have been a real, invisible
   regression against WPF's per-toggle commit.

**Result: 8 of 8 COVERED claims survive.** Two survive with corrected reasoning (`TagsCommitted` describes
dead WPF code; `BindingProxy` carries four bindings, not one), neither of which changes delete-safety.
