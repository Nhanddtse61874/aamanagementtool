# A10 — Controls

**Section risk:** `TeamFilter.ShowTeamColumn` (the signal that is meant to reveal a per-row TEAM
indicator and change grouping once ≥2 teams are checked) is defined in the shared web component but
is not consumed by any of its three screens, and the whole `TeamFilter` control — team-scoping and
TEAM column both — is entirely absent from the web Backlog screen, whose only web request
(`getBacklogList()`) always fetches **every** team the user belongs to with no way to narrow it and
no per-row indicator of which team a backlog belongs to; deleting WPF removes the only place a
multi-team user could currently tell backlogs from different teams apart, or filter the Backlog list
to one team, at all.
