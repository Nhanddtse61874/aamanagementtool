# A4 — Task List (`TaskListTab.xaml` + `TaskListViewModel`, 811 lines)

**Section risk:** The Gantt geometry and every commit path were already ported into `TimesheetApp.Core` / the Angular Task List page before this audit — the worst loss on deletion is silent, cosmetic: two Gantt rendering choices (the full-width "no start date" ghost bar, and the "no dated backlogs" empty-chart caption) and the "Project column visible only in Team-grouped view" affordance quietly disappear with no error and no oracle left to notice, because M10 removes the only app that ever rendered them.

## Method note

`TaskListViewModel.cs` (WPF) is now mostly a thin caller: the Gantt math (`GanttBuilder.BuildGantt`) was already moved verbatim to `src/TimesheetApp.Core/Services/GanttBuilder.cs` (comment at GanttBuilder.cs:5-13 confirms this was an M9 migration, "the web Task List needs the same bars, and a second implementation would be a second source of truth"). The API (`/api/tasklist`, `/api/tasklist/export`) and the Angular page (`task-list.component.ts`/`.html`/`.model.ts`, 723+307+358 lines, plus 739+413 lines of specs) are a near 1:1, deliberately-annotated port of the WPF screen — the Angular source comments explicitly cite WPF line numbers throughout. Every feature below was checked against both sides individually; only three sub-behaviors (all in Gantt/grid presentation, not in data or write paths) failed to find a web equivalent.

## Summary

37 granular features audited. **34 COVERED**, **2 PARTIAL**, **2 MISSING** (all four gaps are presentation-only, confined to Gantt/grid chrome — no write path, no audit trail, and no stored data is lost). Every commit path (inline edits, deadline notes, tag replace-all, task extended/status/tags, Continue) and every piece of business logic (adaptive grouping, chip precedence, working-day Gantt math) was independently verified on both sides and is COVERED with citations. The four non-COVERED items — collapsible bands, the Project label in Team-grouped view, the full-width "unknown start" Gantt ghost, and the "no dated backlogs" Gantt caption — should be logged as a small web follow-up ticket before WPF is deleted, since after deletion there will be no running WPF instance left to compare against and rediscover them.
