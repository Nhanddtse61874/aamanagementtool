## Notes for the parent agent

1. **The one real gap is small and cosmetic.** `.missing__list` needs a `max-height` + `overflow-y:auto`
   to match `ScrollViewer MaxHeight="96"`. Nothing else in A6 is missing. This does not block M10; it is a
   one-line SCSS follow-up.

2. **Checked and found NOT to be gaps** (i.e. the auditor did not miss these):
   - Live cross-tab sync (WPF `DataChangedMessage` → banner + target rebuild) is covered by
     `reports.component.ts:156-161`, though no claim mentioned it.
   - WPF's Week `DatePicker` is not normalised to Monday; the web normalises via `mondayOf`
     (`report-model.ts:106-112`). The web is more correct; no behavior lost.
   - The `teamEmpty()` gate (`reports.component.ts:194-202`) has no WPF counterpart because WPF could send
     a genuinely empty team list. The web renders the empty state locally instead. Equivalent outcome,
     and it prevents a data leak the WPF wire shape never had.

3. **Out of scope but worth flagging elsewhere:** `TeamFilterComponent.reload()`
   (`team-filter.component.ts:146-148`) has **no caller** — the web has no active-team switcher yet, so
   WPF's "switching active team resets each screen's filter" (`TeamFilterViewModel.cs:96-102`) has no web
   trigger. This is a team-switcher gap, not a Reports gap; it belongs to whichever section owns the
   active-team switch.
