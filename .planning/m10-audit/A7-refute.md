## What actually needs a decision before deleting

1. **Avatar colouring is a real regression, not a wash.** The web avatar map has 12 hardcoded names. Every user outside that list — which includes every user an admin creates from now on — is the same green `#0E7C66`. WPF gave 5 deterministic buckets for any name. If the visual differentiation matters, `worklog.service.ts:198-200` needs the char-sum-mod-palette port before the WPF oracle disappears.
2. **Name-only user creation has no web UI.** Confirm the company never needs a roster entry for someone who does not log in (e.g. a contractor whose time is entered by someone else). The API supports it; the screen refuses it. The refusal is deliberate and well-reasoned (the "ghost account" comment at `user-create.ts:6-23`), but it is a capability the WPF app had and the web app does not.
3. Not a defect, just record it: the whole Users screen moved from "any user" to "admin only".
