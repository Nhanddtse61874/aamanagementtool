## Bottom line

- **Survives scrutiny (4):** TaskInputDialog, StandupEntryDialog, DeadlineNoteDialog, TagSelectDialog — all four traced route → wire body → service → component → template on the web side, all genuinely 1:1.
- **Refuted (2):**
  - **StandupIssueDialog → PARTIAL.** Solution and Status are creation-time fields in WPF and hard-coded (`null` / `'open'`) on web; recoverable only via a second edit step.
  - **Common dialog chrome → PARTIAL.** Header/footer bars exist on all five WPF dialogs and on none of the web dialogs; the cited `tag-picker` evidence is a disclosure dropdown, not a modal; Esc/Enter is universal in WPF and present on only 2 of the web dialogs.
- Neither refutation blocks M10 on its own (both are UI-affordance regressions, not data or capability loss), but neither should be recorded as "no change" either — the honest label is *accepted regression*.
