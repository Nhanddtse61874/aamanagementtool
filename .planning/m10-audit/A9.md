# A9 — Dialogs (8) — M10 WPF-Deletion Audit

**Section risk:** The single largest loss risk in this section is Smart Fill — the web dialog is a materially thinner reimplementation (single task, current-week-only days, no "Full 8h" mode, no explicit preview-before-commit step) than WPF's `SmartInputPreviewDialog`, so deleting WPF forever removes the only place a user could multi-task/full-8h/arbitrary-date-range/preview-first smart-fill; everything else in this section (simple input dialogs, tag picker, deadline note) has a verified 1:1 web equivalent and is safe.

## Method note

`SmartInputPreviewDialog`'s own UI orchestration lives in `src/TimesheetApp/ViewModels/SmartInputPanelVm.cs` (dies with WPF), but the actual fill math/validation it calls (`ISmartInputService.BuildPlan`, `ITimeLogService.ValidateSmartFillAsync/ApplySmartFillAsync`) lives in `TimesheetApp.Core` and survives regardless. The verdicts below are about the **dialog's UI surface** (what a user could do through the WPF window), not the underlying Core algorithm, because the question that matters for M10 is "can the user still do this," not "does the math still exist somewhere."

## Summary

- **COVERED (verified 1:1 UI equivalent):** TaskInputDialog, StandupEntryDialog, StandupIssueDialog, DeadlineNoteDialog, TagSelectDialog, common dialog chrome — 6 items.
- **PARTIAL (some sub-behavior lost):** SmartInputPreviewDialog (overall + 5 distinct sub-features: backlog search, multi-task select, Full-8h mode, arbitrary date range, explicit preview-before-commit), QuickImportDialog (source-day picker missing, default-day action kept) — 2 dialogs, 6 lost sub-behaviors total.
- **N/A / already dead:** SelectUserDialog — confirmed fully removed from the codebase already, no risk.

**Recommendation for the M10 decision:** if Smart Fill's multi-task / Full-8h / arbitrary-date-range / preview-first workflow is something real users still rely on, that is genuine functionality loss that should either be built into the Angular Smart Fill dialog before WPF is deleted, or explicitly signed off as an accepted regression — it will not be recoverable from source once `src/TimesheetApp/` is gone.
