# B10 — Theme (light/dark)

**Section risk:** The switching mechanism, persistence, and Settings toggle are solidly replicated in Angular, but the web app's design-token catalogue is materially thinner than WPF's (~21 CSS vars vs ~40 WPF palette keys, and only ~12 of those 21 get an explicit dark override) and at least one page hardcodes hex colors outside the token system — so some screens will not re-color correctly in dark mode, a visual regression with no WPF reference left to catch it once M10 lands.

| Feature | Verdict | Lives in (WPF) | Web evidence | Note |
|---|---|---|---|---|
| Live theme swap with no app restart (palette `ResourceDictionary` swapped in place; all consumers via `{DynamicResource}` re-resolve instantly) | COVERED | `src/TimesheetApp/Services/ThemeService.cs:15-37`, `IThemeService.cs:6-10` | `src/timesheet-web/src/app/services/theme.service.ts:17-27` (Angular `effect()` toggles `dark` class on `<html>`) + `src/timesheet-web/src/styles.scss:8-43` (`:root` / `.dark` CSS custom properties cascade live, no reload) | [VERIFIED] Different mechanism (CSS cascade vs XAML resource-dictionary swap), same observable behavior: instant re-theme, no restart. |
| Theme preference persisted across sessions, applied before first paint (no flash of wrong theme) | COVERED | `src/TimesheetApp/App.xaml.cs:46-48` (theme applied before any window renders, read from `IAppConfig.IsDarkMode`), `src/TimesheetApp/ViewModels/SettingsViewModel.cs:147-177` (`OnIsDarkModeChanged` persists via `_config.SetIsDarkMode`) | `src/timesheet-web/src/main.ts:5-8` (pre-paint: reads `localStorage['worklog.theme']` and adds `.dark` before Angular bootstraps) + `theme.service.ts:1-27` (persists `{dark, accent}` to `localStorage` on every change) | [VERIFIED] Same "avoid flash" pattern, same persistence intent. Recommend a human pass over each page in dark mode before WPF is deleted — this is exactly the kind of gap that has no oracle left to catch once the WPF reference app is gone. |
