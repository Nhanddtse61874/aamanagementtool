## Summary

All 9 rules in this section are implemented in `TimesheetApp.Core/Services/TimeLogService.cs` (plus `SmartInputService.cs` for the smart-fill variants and `TimeLogRepository.cs` for the upsert SQL) and are reachable from the web app through the API's checked write path (`PUT`/`DELETE /api/timesheet/cell`, wired in `TimesheetApp.Api/Endpoints/TimesheetEndpoints.cs:84,119`), which the code's own comments confirm is "the web API's write path" (`TimeLogService.cs:65-75`). The only WPF-side artifacts found (`TimesheetRowVm.cs:51-62`) are a client-side pre-validation echo used for instant UI feedback — cosmetic, not authoritative. Deleting `src/TimesheetApp/` cannot lose any of these 9 behaviors; the DayCap rule set is entirely CORE-SURVIVES.
