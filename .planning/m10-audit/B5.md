## Summary

Every algorithmic rule in this section (`ScheduleStateService.Evaluate`'s nine steps, `WorkingDayCalculator`, `BacklogContinuationService`'s full clone pipeline, `SetTagsAsync`'s replace-all + single-audit-row guarantee) lives in `TimesheetApp.Core` and is unreachable-by-deletion under M10 as scoped (only `src/TimesheetApp/` + its ViewModels/Views/App.xaml.cs die). All of it is additionally already wired into real API routes (`/api/tasklist`, `/api/backlogs/{id}/continue`, `/api/backlogs/{id}/tags`, `/api/holidays`) and rendered/edited from Angular (`task-list.component.ts`, `task-list.model.ts`, `holiday-calendar.ts`, `settings.component.ts`), with the Late/Warning precedence and the "0 tasks ≠ Done" trap both explicitly re-documented (not silently re-derived) on the Angular side.

Two sub-clauses are flagged [ASSUMED] rather than [VERIFIED] and should get a follow-up look (not blocking, since the underlying Core logic survives regardless):
1. The Backlog-editor's progress-field-disabled state (only the Task-List inline-edit half was directly confirmed).
2. Log Work's holiday **column** rendering in the Angular grid (only the holiday-blocks-validation effect was confirmed, not a visible column).
