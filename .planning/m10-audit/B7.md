# B7 ‚Äî Export / ExportHub / SharePoint

**Section risk:** the only behavior at real risk of silent loss is the automatic 12-month/12-week
catch-up backfill (`ExportHubService.BackfillAsync`) ‚Äî its Core method survives deletion, but the
WPF-only startup trigger that called it dies with no scheduler/hosted-service replacement in the
API, so on a web-only deployment historical export folders never self-heal after a gap; everything
else in this section (Excel/Markdown export, the mirrored two-root folder tree, per-root
best-effort, segment dedupe, and the SharePoint destination validation rules) is proven to run
through the same Core code path from both the WPF and web callers.
