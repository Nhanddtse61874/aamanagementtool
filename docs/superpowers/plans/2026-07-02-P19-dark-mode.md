# P19 — Dark Mode — PLAN (wave-grouped)

**Goal:** An instant, no-restart dark-mode toggle in Settings (idiomatic WPF: palette split + `DynamicResource` + dictionary swap), persisted in config, applied on startup — light mode unchanged, verified in both themes.
**Mode A.** Spec: `docs/superpowers/specs/2026-07-02-dark-mode-design.md`. **Stack skill: `dotnet`.**
**Tech Stack:** .NET 8 / WPF / ResourceDictionary theming / CommunityToolkit.Mvvm / xUnit STA render guards.

## must_haves (goal-backward)
**Observable Truths** — OT-1 Settings toggle switches the whole app instantly (no restart). OT-2 Choice persists (appsettings.json) + applied on startup. OT-3 Every surface dark in dark mode (no light patch). OT-4 Light mode unchanged. OT-5 No render-crash either theme. OT-6 Build clean, full suite green.

**Required Artifacts:** `Views/Theme/Palette.Light.xaml`, `Palette.Dark.xaml`; refactored `Theme.xaml` (palette removed, styles use DynamicResource, literals promoted); `App.xaml`(+`.cs`) merge/startup; `Services/IThemeService.cs`+`ThemeService.cs`; `Config/IAppConfig.cs`+`JsonAppConfig.cs` (`IsDarkMode`); `SettingsViewModel`+`SettingsTab.xaml` toggle; all `Views/**/*.xaml` palette-ref conversion; render tests (both palettes).
**Key Links:** config `IsDarkMode` → `ThemeService.Apply` (startup + toggle) → swap palette dict → `DynamicResource` consumers (Theme styles + views) re-resolve.

## Risks / gates
1. **Over-conversion** — scripted, key-list-driven; grep reconciliation gate (W2). Style/type refs untouched.
2. **Missed literals** — post-W2 hex audit over Views (minus shadows).
3. **Render matrix** — both-theme STA guards (W3) are the primary safety net for the render-crash class.
4. **Gantt canvas** — code-behind reads brushes via `TryFindResource` at draw; picks up the swap on next redraw. Note; optionally force-redraw on theme change (W3).
5. **Render-test harness** now needs a palette dict merged (W1 updates `EnsureAppResources`).
6. **Build/run gotcha:** kill running `TimesheetApp` before build.
